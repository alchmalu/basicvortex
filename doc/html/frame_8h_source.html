309  // A frame is as a new coordinate system, defined with respect to a reference frame (the world
310  // coordinate system by default, see the "Composition of frame" section).
311 
312  // The transformOf() (resp. inverseTransformOf()) functions transform a 3D vector from (resp.
313  // to) the world coordinates system. This section defines the 3D vector transformation
314  // functions. See the Coordinate system transformation of 3D points above for the transformation
315  // of 3D points. The difference between the two sets of functions is simple: for vectors, only
316  // the rotational part of the transformations is taken into account, while translation is also
317  // considered for 3D points.
318 
319  // The length of the resulting transformed vector is identical to the one of the source vector
320  // for all the described functions.
321 
322  // When local is prepended to the names of the functions, the functions simply transform from
323  // (and to) the reference frame.
324 
325  // When In (resp. From) is appended to the names, the functions transform from (resp. To) the
326  // frame that is given as an argument. The frame does not need to be in the same branch or the
327  // hierarchical tree, and can be \c NULL (the world coordinates system).
328 
329  // Combining any of these functions with its inverse (in any order) leads to the identity.
